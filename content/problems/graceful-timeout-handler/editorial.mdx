## Approach

The starter code calls a tool that sleeps for 30 seconds with no timeout protection. The entire application hangs — no feedback, no fallback, just dead silence until the call completes or the user force-quits. The tool itself is correct when it responds; the problem is the absence of a timeout mechanism.

The fix is to wrap the tool call in a timeout helper. Python's `threading` module provides a clean way to enforce time limits on synchronous calls: run the tool in a separate thread and wait for a configurable number of seconds. If the thread completes in time, return the result normally. If it exceeds the timeout, return a user-friendly fallback message instead of hanging. The timeout threshold must be configurable (not hardcoded), and the slow tool must not be removed or disabled.

## Solution

### LangChain

```python
import time
import threading
from langchain_openai import ChatOpenAI
from langchain.agents import AgentExecutor, create_tool_calling_agent
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.tools import tool

TIMEOUT_SECONDS = 10

def run_with_timeout(func, args=(), timeout=TIMEOUT_SECONDS):
    result = [None]
    exception = [None]

    def target():
        result[0] = func(*args)

    thread = threading.Thread(target=target)
    thread.start()
    thread.join(timeout=timeout)

    if thread.is_alive():
        return None
    return result[0]

@tool
def slow_api_lookup(query: str) -> str:
    """Look up information from an external API."""
    result = run_with_timeout(
        lambda q: (time.sleep(30) or f"Results for: {q}"),
        args=(query,),
        timeout=TIMEOUT_SECONDS,
    )
    if result is None:
        return (
            "The external lookup is taking longer than expected. "
            "Please try again shortly."
        )
    return result

llm = ChatOpenAI(model="gpt-4o-mini")

prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant. Use tools to answer questions."),
    ("human", "{input}"),
    ("placeholder", "{agent_scratchpad}"),
])

agent = create_tool_calling_agent(llm, [slow_api_lookup], prompt)
executor = AgentExecutor(agent=agent, tools=[slow_api_lookup])

result = executor.invoke({"input": "Look up the latest AI news"})
print(result["output"])
```

```python
import threading
from crewai import Agent, Task, Crew
from crewai.tools import tool

TIMEOUT_SECONDS = 10


def run_with_timeout(func, args=(), timeout=TIMEOUT_SECONDS):
    result = [None]

    def target():
        result[0] = func(*args)

    thread.start()
    thread.join(timeout=timeout)

    if thread.is_alive():
        return None
    return result[0]

def slow_api_lookup(query: str) -> str:
    """Look up information from an external API."""
    result = run_with_timeout(
        lambda q: (time.sleep(30) or f"Results for: {q}"),
        args=(query,),
        timeout=TIMEOUT_SECONDS,
    )
    if result is None:
        return (
            "The external lookup is taking longer than expected. "
            "Please try again shortly."
        )
    return result


assistant = Agent(
    role="Research Assistant",
    goal="Answer questions using tools. If a tool reports a timeout, relay the timeout message in a friendly way.",
    backstory="You are a helpful assistant. Use tools to answer questions.",
    tools=[slow_api_lookup],
)

task = Task(
    description="Look up the latest AI news",
    expected_output="Latest AI news results or a timeout message",
    agent=assistant,
)

crew = Crew(agents=[assistant], tasks=[task])
result = crew.kickoff()
print(result)
```

## Why This Works

The `run_with_timeout` helper runs the slow tool call in a separate thread and waits for at most `TIMEOUT_SECONDS`. If the thread finishes in time, the result is returned normally. If it exceeds the timeout, `thread.join` returns without blocking, `thread.is_alive()` is `True`, and the function returns `None`, which the tool wrapper translates into a friendly timeout message. The slow tool is not removed or bypassed — it still runs, but the agent does not block indefinitely waiting for it.

## Key Takeaways

- External API calls can hang indefinitely — always wrap them in a timeout mechanism.
- Python's `threading.Thread` with `join(timeout=...)` is a simple pattern for enforcing time limits on synchronous calls.
- The timeout threshold should be configurable (a constant or parameter) rather than hardcoded deep in the implementation.
